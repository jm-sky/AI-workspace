**Data:** 2026-07-20

**Wzorzec Kancelarii:** [#075](../../../ai-kancelaria/docs/alfa/issues/2026-075-tool-search-tool.md) (TF-IDF pre-filter) — u nas cel docelowy bliższy Anthropic Tool Search Tool

## Cel

Gdy zestaw narzędzi rośnie, nie ładować całego katalogu do każdej tury.

**Model listy narzędzi:**
1. **Narzędzia podstawowe** — zawsze dostępne (m.in. meta-narzędzie *tool search*).
2. **Narzędzia dedykowane agenta** — z profilu / wybranego agenta (kontekst routingu).
3. **Gdy narzędzi jest mało** (poniżej progu) — można przekazać **wszystkie** bez search.

Agent może wołać tool search, gdy potrzebuje narzędzia spoza aktualnie załadowanego podzbioru.

## Stan obecny (Workspace)

- Pełny profil tooli per `agent_key` (`AGENT_TOOL_PROFILES`) + sekcja `AVAILABLE TOOLS` w promptcie.
- Brak meta-narzędzia search / defer_loading.
- Przy małej liczbie tooli (github + memory) problem jeszcze nie boli — ale blokuje skalę i routing wielu agentów.

## Stan Kancelarii (po pullu 2026-07)

- ✅ `tool_search.py` — TF-IDF in-memory, `filter_tools_for_query()` gdy `len(tools) > TOOL_SEARCH_THRESHOLD` (15).
- Uproszczenie: **filtr przed pętlą**, nie osobne narzędzie wywoływane przez model.
- Domyślnie wyłączone (`TOOL_SEARCH_ENABLED=false`).
